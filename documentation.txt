1. OVERVIEW
--------------------------------------------------------------------------------
OpticsCore is a C++17 desktop application providing 50 interactive optics
simulations with real-time visualization. The physics engine is decoupled from
the GUI, allowing standalone use as a static library or via the C API.

Technology stack:
  - Language:      C++17
  - Build system:  CMake 3.15+ (Ninja or MSVC generators)
  - GUI:           Dear ImGui v1.91.8, ImPlot v0.16
  - Windowing:     GLFW 3.4, OpenGL 3.3 core profile
  - Platform:      Windows (MSVC), Linux (GCC/Clang), macOS (Clang)


2. PROJECT STRUCTURE
--------------------------------------------------------------------------------

  File                      Purpose
  ────────────────────────  ──────────────────────────────────────────────────
  CMakeLists.txt            Root build file; FetchContent for dependencies
  OpticalSystem.h/.cpp      Physics engine — all 50 computations
  OpticsConstants.h         SI physical constants (inline constexpr)
  OpticsExport.h            DLL/shared-library export macros (OPTICS_API)
  NumericalSolverOpt.h/.cpp Numerical methods (BPM step, Gaussian beam init)
  optics_c_api.h/.cpp       C-compatible wrapper API
  GuiApp.h/.cpp             ImGui/ImPlot GUI: sidebar, parameters, plotting
  main.cpp                  Application entry: GLFW window, render loop
  pch.h                     Precompiled header (MSVC only)

Build targets:
  OpticsPhysics  — static library (physics only, no GUI dependencies)
  OpticsCore     — executable (links OpticsPhysics + ImGui + ImPlot + OpenGL)

5. C API REFERENCE
--------------------------------------------------------------------------------
The C API (optics_c_api.h) provides language-agnostic access to the physics
engine. All functions take an opaque OP_Handle as the first argument.

  Lifecycle:
    OP_Handle op_create(name, wavelength, n_medium)
    void      op_destroy(handle)

  Getters:
    op_get_wavelength(h)
    op_get_n_medium(h)

  Physics functions follow the pattern:
    op_<module>_<function>(handle, ...)

  Examples:
    op_snell_refraction(h, theta_i, n1, n2)
    op_thin_lens(h, obj_dist, f)
    op_airy_intensity(h, theta, D, lambda)
    op_fiber_na(h, n_core, n_clad)

7. BUILD INSTRUCTIONS
--------------------------------------------------------------------------------

  Prerequisites:
    - CMake >= 3.15
    - C++17 compiler (MSVC 2019+, GCC 9+, Clang 10+)
    - OpenGL 3.3 capable GPU/driver
    - Git (for FetchContent dependency download)

  Quick build (Ninja):
    cmake -B build -G Ninja
    cmake --build build
    ./build/OpticsCore

  MSVC (Visual Studio):
    cmake -B build -G "Visual Studio 17 2022"
    cmake --build build --config Release
    .\build\Release\OpticsCore.exe

  All dependencies (GLFW, Dear ImGui, ImPlot) are downloaded automatically
  via CMake FetchContent. No manual dependency installation required.

  MSVC-specific flags applied automatically:
    - _USE_MATH_DEFINES (for M_PI)
    - NOMINMAX (prevent min/max macro conflicts)
    - Precompiled headers (pch.h) for OpticsCore target


8. CSV EXPORT
--------------------------------------------------------------------------------
Most simulations provide an exportXxxCSV(...) function that writes computed
data to a CSV file for external analysis (MATLAB, Python, Excel, etc.).
Each CSV includes a header row with column names and uses comma separation.
